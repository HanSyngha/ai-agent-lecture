planning_system_prompt = """당신은 채용 업무 실행 계획을 수립합니다. 규칙: 1. 실행 전에 전체 계획을 먼저 완성. 2. 평가기준표는 하나만 조회. 3. 이력서 비교가 필요하면 모든 이력서를 먼저 조회 후 평가. 출력 형식 (JSON만): { "goal": "목표 요약", "plan": [ {"step": 1, "tool": "툴이름", "desc": "이 단계에서 할 일"}, {"step": 2, "tool": "툴이름", "desc": "..."} ] }""" def make_plan(goal): messages = [{"role": "system", "content": planning_system_prompt}, {"role": "user", "content": goal}] message = generate_response(messages, tools) return json.loads(message.content)

Code · execute_plan 골격

각 step 안에
작은 피드백 루프.

def execute_plan(plan, goal): history = [{"role": "user", "content": f"목표: {goal}\n계획: {plan['plan']}"}] for step_info in plan["plan"]: step_num = step_info["step"] desc = step_info["desc"] for inner in range(5): messages = [{"role": "system", "content": execution_system_prompt}, *history, {"role": "user", "content": f"Step {step_num}만 실행: {desc}"}] message = generate_response(messages, tools, "auto") if not message.tool_calls: # 이 step 완료 history.append({"role": "assistant", "content": message.content}) break history.append({"role": "assistant", "tool_calls": [...]}) for tc in message.tool_calls: result = TOOLS[tc.function.name](**json.loads(tc.function.arguments)) history.append({"role": "tool", "tool_call_id": tc.id, "content": ...})

외곽 for문 = 계획대로 step 순회. 내부 for문 = step 안에서 tool_calls 반복(작은 피드백 루프). step 종료 조건 = tool_calls 없음.

Chapter 9 · 마무리

한 LLM이
전부 짊어진다.
그게 한계다.

평가 규칙, 저장 규칙, 보고서 규칙, 엑셀 컬럼명 등 모든 게 한 시스템 프롬프트에 쌓입니다. 컨텍스트가 길어질수록 LLM이 헷갈리기 시작합니다. 마지막 챕터에서 역할별로 에이전트를 쪼갭니다.

CH 10 → 멀티 에이전트 하나의 거대한 LLM이 아니라, 역할별 작은 에이전트들의 협업.
