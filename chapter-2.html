키·URL·이름,
한 번만
정리해두기.

실습마다 인증키와 모델명과 base URL을 반복 입력하면 코드가 더러워집니다. .env 한 파일에 모아두고, 모든 셀이 거기서 꺼내 씁니다.

Anti-pattern → pattern

매 셀마다 키 입력?
한 번이면 충분합니다.

❌ 매 셀마다
# Cell 1 api_key = "sk-..." url = "https://..." # Cell 2 (같은 거 또) api_key = "sk-..." url = "https://..."
✅ .env 한 번
# .env 파일 (한 번만) api_key=sk-... api_base_url=https://... # 어느 셀에서든 from dotenv import load_dotenv load_dotenv() key = os.getenv("api_key")

.env는 git에 올리지 않는 게 원칙. .gitignore에 .env를 추가해 두세요.

← Hub03
Code · .env 만들기

dict 한 번 →
.env 파일 한 번.

import os from dotenv import set_key env_vars = { "OPENAI_API_KEY": OPENAI_API_KEY, "credential_key": credential_key, "send_system_name": send_system_name, "model": model, "api_base_url": api_base_url, "user_id": user_id, "embedding_credential_key": embedding_credential_key, "user_pw": user_pw, } for key, value in env_vars.items(): set_key(".env", key, value)

set_key는 .env가 없으면 만들고, 있으면 해당 키만 갱신합니다. 비밀번호처럼 한 번만 받아두면 끝.

← Hub04
Code · 매 노트북의 첫 두 줄

한 번 적고,
매 노트북에서
두 줄로 불러온다.

load_dotenv(override=True)로 환경변수를 프로세스에 주입하고, os.getenv()로 어디서든 꺼냅니다.

import os from dotenv import load_dotenv load_dotenv(override=True) # 어디서든 꺼내 쓰기 key = os.getenv("credential_key") url = os.getenv("api_base_url") name = os.getenv("model")
