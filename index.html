Chapter Index · 클릭해서 이동
CH 01 환경 셋업 파이썬 패키지, 기존 웹앱 종료. 강의 실습 환경의 출발선. CH 02 환경변수 .env 매번 반복되는 키·URL·시스템명을 .env로 한 번만. 클라이언트 만들 준비. CH 03 시스템 프롬프트 & 페르소나 generate_response()의 뼈대, system vs user, 스트리밍, 역할 부여. CH 04 멀티턴 & 히스토리 LLM이 이전 대화를 기억하지 못하는 이유와 history 리스트의 역할. CH 05 툴 정의와 개별 사용 DB, RAG, 위키, 엑셀, 메일 — LLM의 손과 발을 하나씩 직접 호출. CH 06 고정 워크플로우 툴 → 툴 → 툴. 코드로 순서를 박아두면 한 시나리오만 잘 돈다. CH 07 Decision · 자율 툴 선택 시스템 프롬프트로 툴 목록을 알려주고, JSON으로 무엇을 호출할지 답하게 한다. CH 08 피드백 루프 · Tool Calling API 레벨 tool_calls와 ReAct(생각·행동·관찰)의 자동 반복. CH 09 Planning 실행 전에 전체 계획을 먼저 짠다. 그리고 계획대로만 움직인다. CH 10 멀티 에이전트 오케스트레이터가 역할별 에이전트를 호출. 하나의 LLM이 모든 걸 짊어지지 않는다.
